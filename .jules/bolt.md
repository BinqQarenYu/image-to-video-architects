## 2026-03-05 - Parallel PIL Image Processing in FastAPI
**Learning:** Sequential image processing (resizing, converting) in FastAPI endpoints blocks the event loop for CPU-bound tasks and waits synchronously for I/O, leading to significant delays when processing multiple assets. Using `asyncio.to_thread` with `asyncio.gather` allows offloading these tasks to a thread pool, enabling parallel execution without blocking the main event loop. Additionally, switching from `LANCZOS` to `BILINEAR` resizing provides a substantial speed boost for video frame preparation where micro-details are less critical.
**Action:** Use `asyncio.gather(*[asyncio.to_thread(func, *args) for ...])` for parallelizing heavy synchronous asset processing in async endpoints. Prefer `BILINEAR` over `LANCZOS` for video generation frames to prioritize speed.
